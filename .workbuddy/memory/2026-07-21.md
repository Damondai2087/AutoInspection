## Windows 巡检排障（22:2x）
- 实测：本机 Windows 巡检可成功（exit 0，生成 ~37KB UTF-8 HTML 报告）。用户感知的"有问题"实为两点：
  1. 日志面板在巡检过程中空白：Python 对管道 stdout 块缓冲，脚本输出直到进程退出才一次性刷出；且后端原用 BufReader::lines() 按 UTF-8 读，GBK 字节会整行丢弃。
  2. 报告预览 iframe 原 sandbox 仅 allow-same-origin，拦了报告内 script，控制台报 "Blocked script execution in about:srcdoc"。
- 修复：spawn_reader 改字节级读取 + decode_line（UTF-8 优先失败 lossy）；run_local_windows/run_command 增加诊断日志（Python 解释器/命令/工作目录/退出码/报告路径）；ReportPreview.vue sandbox 改 allow-scripts allow-same-origin。
- 验证：cargo check 通过；pnpm build 通过。
